# Задание 2.2 E2E тесты таск-трекера.

Базовый URL: https://avito-tech-internship-psi.vercel.app/issues

Страница списка задач: /issues

## 1. Создание задачи

### TC_E2E_01 - Успешное создание задачи с вводом обязательных полей

**Цель:** проверить, что задача создается только при заполнении обязательных полей

**Условия:**
* Открыта страница /issues

**Шаги:**
* Нажать кнопку «Создать задачу»
* Ввести в поле «Название» текст/Test issue
* Ввести в обязательные поля текст
* Не заполнять необязательные поля
* Нажать кнопку «Создать»

**Ожидаемый результат:**
* Окно создания задачи закрывается
* В списке задач появляется строка с названием Test issue
* У задачи стоит дефолтный статус (например Backlog)

### TC_E2E_02 - Успешное создание задачи с заполнением всех полей

**Цель:**
проверить, что все введение поля сохраняются.

**Условия:**
* Открыта страница /issues

**Шаги:**
* Нажать кнопку «Создать задачу»
* Заполнить поле «Название» текст/Test issue
* Заполнить поле «Описание» текст/произвольный
* Заполнить другие поля «Проект», «Приоритет», «Статус», «Исполнитель»/произвольно
* Нажать кнопку «Создать»

**Ожидаемый результат:**
* Окно создания задачи закрывается
* В списке задач появляется строка с названием Test issue
* У задачи стоит дефолтный статус
* При открытии карточки задачи отображаются все введённые поля

### TC_E2E_03 - Валидация: создание задачи с пустым названием

**Цель:**
убедиться, что обязательное поле «Название» валидируется.

**Шаги:**
* Нажать кнопку «Создать задачу»
* Оставить обязательное поле «Название» пустым
* Нажать «Создать»

**Ожидаемые результат:**
* Задача не создается
* Появляется сообщение об ошибке у поля «Название»
* Кнопка «Создать» остается доступной, но надо исправить ошибку
* Кнопка «Создать» неактивна, если поле «Название» пустое

### TC_E2E_04 - Отмена создания задачи

**Цель:**
проверить, что при заполнении полей и отмене создания задачи, новая задача не появляется в списке

**Шаги:**
* Нажать «Создать задачу»
* Ввести текст в поле «Название» текст/Test issue
* Ввести произвольный текст в обязательные и не обязательные поля
* Отменить заполнения формы

**Ожидаемый результат:**
* Форма создания задач закрывается
* В списке задач не появляется задача с названием Test issue

### TC_E2E_05 - Создание двух задач с одинаковым названием

**Цель:**
проверить поведение сайта при создании дублированных тем

**Шаги:**
* Нажать кнопку «Создать задачу»
* Ввести в поле «Название» текст/Test issue
* Заполнить все необходимые поля
* Нажать кнопку «Создать»
* Нажать повторно кнопку «Создать задачу»
* Ввести в поле «Название» такой же текст как и в первой задаче/Test issue
* Дублировать остальные поля
* Нажать «Создать»
**Ожидаемый результат:**
* Обе задачи отображаются в списке, при условии, что дуликаты можно создавать
* Либо система выдает понятную ошибку при создании второй задачи
* В любом случае поведение системы понятно и не вызывает багов

## 2. Открытие карточки задачи

### TC_E2E_06 - Открытие карточки задачи из списка

**Условия:**
* Открыт список созданных задач

**Цель:**
проверить фунцию открытия созданной карточки, а также проверка, все ли заполненные данные отображаются

**Шаги:**
* Открыть список созданных задач
* Найти в списке нужную задачу
* Кликнуть по карточке созданного задания

**Ожидаемый результат:**
* Открывается карточка задачи
* В карточке отображаются все заполненные поля

### TC_E2E_07 - Закрытие карточки задачи

**Цель:**
проверить, что карточка задачи закрывается корректно и пользователь возвращается к списку задач

**Шаги:**
* Выполнить шаги из раздела TC_E2E_06
* Нажать кнопку закрытия задачи

**Ожидаемый результат:**
* Карточка закрывается
* Пользователь возвращается к списку задач
* Положение пользователя на странице сохраняется после закрытия карточки

### TC_E2E_08 - Изменение данных в созданной карточке задачи

**Цель:** 
проверить, что изменения в карточке отражаются в списке.

**Условия:**
существует карточка для редактирования данных

**Шаги:**
* Открыть карточку задачи из списка созданных задач
* Изменить название задачи
* Сохранить изменения
* Закрыть карточку

**Ожидаемый результат:**
* В списке задач отображается новое название отредактированной карточки

## 3. Поиск задачи

### TC_E2E_09 - Поиск по точному названию

**Условия:**
Есть сущствуюшие карточки задач

**Цель:**
базовый позитивный сценарий поиска.

**Шаги:**
* Открыть список созданных задач
* В поле поиска ввести название существующей карточки
* Начать поиск

**Ожидаемый результат:**
* В результате поиска отображается искомая карточка задачи
* Другие карточки не отображаются

### TC_E2E_10 - Поиск по части названия

**Цель:** 
проверить, какой результат выдаст сайт при неполном вводе

**Шаги:**
* Ввести в поиск часть названия существующей карточки, допустим Test
* Начать поиск

**Ожидемый результат:**
* В списке задач появились только те задачи, в названии которых есть слово Test

### TC_E2E_11 - Поиск без результатов

**Шаги:**
* Ввести в поле поиска название несуществующей задачи
* Начать поиск

**Ожидаемый результат:**
* В списке нет задач
* Отображается надпись «Задачи не найдены»

## 4. Переход на доску проекта

### TC_E2E_12 - Переход на доску из верхнего меню / ссылки на проект

**Цель:**
проверить навигацию на доску проекта

**Шаги:**
* Открыть список созданных задач
* В шапке или рядом со списком задач нажать ссылку/кнопку «Проекты»

**Ожидаемый результат:**
* Открывается страница проектов
* Отображается список проектов

### TC_E2E_13 - Возврат с вкладки проектов к списку задач

**Шаги:**
* Открыть список проектов из вкладки списка задач
* Нажать любой элемент возврата к прошлой странице или кнопку списка задач в интерфейсе

**Ожидаемый результат:**
* Открывается список созданных задач
