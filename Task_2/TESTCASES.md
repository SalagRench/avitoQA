# Тест-кейсы: Task Tracker (E2E)

Базовый URL: `https://avito-tech-internship-psi.vercel.app`
Страница списка задач: `/issues`

## 1. Создание задачи

### TC_E2E_01 — Успешное создание с обязательными полями
**Цель:** задача создаётся при заполнении только обязательных полей.  
**Условия:** открыта `/issues`.  
**Шаги:** нажать «Создать задачу» → заполнить «Название» (например, `Test issue`) → не заполнять необязательные поля → «Создать».  
**Ожидания:** модалка закрыта; в списке есть `Test issue`; статус по умолчанию (Backlog).

### TC_E2E_02 — Создание с заполнением всех полей
**Цель:** все введённые поля сохраняются.  
**Условия:** открыта `/issues`.  
**Шаги:** «Создать задачу» → заполнить «Название» `Test issue` → «Описание» произвольно → «Проект», «Приоритет», «Статус», «Исполнитель» произвольно → «Создать».  
**Ожидания:** модалка закрыта; в списке есть `Test issue`; при открытии карточки видно все введённые поля.

### TC_E2E_03 — Валидация пустого названия
**Цель:** поле «Название» обязательно.  
**Шаги:** «Создать задачу» → оставить «Название» пустым → «Создать».  
**Ожидания:** задача не создаётся; отображается ошибка у «Название»; при пустом поле кнопка «Создать» не активна или повторная попытка требует исправления.

### TC_E2E_04 — Отмена создания
**Цель:** отмена не добавляет задачу.  
**Шаги:** «Создать задачу» → ввести «Название» `Test issue` + произвольные поля → закрыть модалку (клик вне/Отмена).  
**Ожидания:** модалка закрыта; в списке нет `Test issue`.

### TC_E2E_05 — Дубликат названия
**Цель:** поведение при одинаковых названиях.  
**Шаги:** создать задачу `Test issue`; создать вторую с тем же названием и полями.  
**Ожидания:** обе задачи отображаются или показана понятная ошибка; поведение консистентно.

## 2. Открытие карточки задачи

### TC_E2E_06 — Открытие карточки из списка
**Цель:** карточка открывается и содержит данные.  
**Условия:** есть созданная задача.  
**Шаги:** найти задачу в списке → клик по карточке.  
**Ожидания:** карточка открыта; отображаются все заполненные поля.

### TC_E2E_07 — Закрытие карточки
**Цель:** корректное закрытие.  
**Шаги:** открыть карточку (TC_E2E_06) → нажать закрытие.  
**Ожидания:** карточка закрыта; возврат к списку задач; позиция в списке сохранена.

### TC_E2E_08 — Редактирование карточки
**Цель:** изменения сохраняются.  
**Условия:** существует задача для редактирования.  
**Шаги:** открыть карточку → изменить «Название» → сохранить → закрыть.  
**Ожидания:** в списке отображается новое название.

## 3. Поиск задачи

### TC_E2E_09 — Поиск по точному названию
**Условия:** есть задачи.  
**Шаги:** ввести точное название задачи в поиск → начать поиск.  
**Ожидания:** отображается только искомая карточка.

### TC_E2E_10 — Поиск по части названия
**Шаги:** ввести часть названия (например, `Test`) → начать поиск.  
**Ожидания:** показаны задачи, где в названии есть `Test`.

### TC_E2E_11 — Поиск без результатов
**Шаги:** ввести несуществующее название → начать поиск.  
**Ожидания:** список пуст; показано «Задачи не найдены».

## 4. Переход на доску проекта

### TC_E2E_12 — Переход на доску из меню/линка
**Цель:** навигация на страницу проектов.  
**Шаги:** на `/issues` нажать «Проекты»/ссылку на доску.  
**Ожидания:** открыта страница проектов; отображён список проектов.

### TC_E2E_13 — Возврат со страницы проектов
**Шаги:** на странице проектов нажать элемент возврата/кнопку списка задач.  
**Ожидания:** открыт список задач.

## 5. Фильтры

### TC_E2E_14 — Фильтр по статусу Backlog
**Шаги:** открыть `/issues` → выбрать в фильтре статусов `Backlog`.  
**Ожидания:** видны только задачи со статусом Backlog.

### TC_E2E_15 — Фильтр по доске
**Шаги:** открыть `/issues` → выбрать доску в фильтре «Доска».  
**Ожидания:** видны только задачи выбранной доски.
